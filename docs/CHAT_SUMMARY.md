# 🗨️ 開発チャット・意思決定ログ (2025-12-24)

本ドキュメントは、AIビジネス戦略シュミレーターの開発における主要な議論、意思決定、および実装されたプロンプト思考ロジックの要約です。

## 1. 核心的な意思決定

### 統合型AIアドバイザーへの移行
*   **課題**: フローティングドロワー（引き出し型）では、分析結果を見ながらチャットするのが難しく、画面の一部が隠れてしまう。
*   **決定**: ダッシュボードの最下部に常駐する「フルワイド・セクション」として統合。
*   **効果**: ユーザーは分析データ（PL、ターゲット、課題）を俯瞰しながら、そのままAIと壁打ちができるようになり、UXが「ツール」から「対話型プラットフォーム」へ進化した。

### 視覚的バリデーション（M00教育）
*   **課題**: 準備が整っていない状態で「詳細生成」を押すと、AIが不完全な推測をしてしまう。アラートだけではユーザーのネクストアクションが不明確。
*   **決定**: アラートと同時に、サイドバーの「構造化整理（M00）」モジュールを**物理的に点滅（Pulse）**させる機能を実装。
*   **効果**: 「何が足りないか」を視覚的にガイドすることで、ユーザーの離脱を防ぎ、正しいプロセス（構造化→詳細化）へ誘導することに成功。

### M12（トレンド分析）の独立実装
*   **課題**: 市場分析（M10）の中に入れ込むと情報過多になり、SNSや記事作成（M50）の文脈でのトレンド把握が弱くなる。
*   **決定**: トレンド・キーワードに特化したM12モジュールを単独で実装。
*   **技術**: プロンプトにプラットフォーム別の「攻略アプローチ」を含め、単なるデータ出しではない「戦略的提言」を行うように調整。

## 2. 実装された重要ロジック（思考アルゴリズム）

### モジュール選定スコアリング
特定のキーワードに基づいて、AIが分析に必要と判断するモジュールを動的に決定。
*   `売上` `集客` → **M20 (販売戦略)**
*   `アプリ` `画面` `要件` → **M60/M61 (アプリ開発・仕様書)**
*   `融資` `事業計画` → **M30/M31 (計画・資金調達)**

### AIコンテキスト接続 (Analysis Context Connected)
`chatWithAI` 関数において、現在の全分析データ（JSON）を `system_instruction` に近い形で履歴の先頭に注入。これにより、AIはユーザーが編集した最新の数値を前提に会話ができる。

## 3. 今後の戦略的拡張案

1.  **リアルタイム共同編集モード**: 複数のタブやデバイスでの状態同期。
2.  **業界別DB接続**: トレンド分析（M12）において、実際の検索API等との連携検討。
3.  **多言語展開**: 現在は日本語ファーストだが、インバウンドや海外進出向けの英語化対応。
